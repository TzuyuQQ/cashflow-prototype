第一部分 — 快速入门
工具简介
这个工具是什么？它为什么被开发出来？
工具是什么

现金流管理工具是一个为 PMO 设计的项目资金监控平台。它将所有项目的收支数据集中在一处，帮助 PMO 实时掌握各项目的资金状况，并支持按项目经理维度进行快速筛选和分析。

它解决了什么问题
  • 数据分散：原本每个项目各有一套 Excel，合并费时。本工具将全部数据集中管理，一个界面看全局。
  • 风险发现滞后：逾期未收款、资金缺口往往发现太晚。工具提供状态标记和提醒，帮助提前预警。
  • 无法追踪单个 PM：PMO 过去无法快速查看某一位项目经理的资金情况。筛选器功能解决了这个问题。
  • 缺乏预测视图：没有向前看的工具。月度预测模块提供未来12个月的滚动现金流预计。
适用人员
PMO 管理人员
监控全部项目组合的整体现金流；按项目经理维度追踪各 PM 负责项目的回款情况。
项目经理（PM）
查看自己负责项目的收支状态、应收账款账龄、以及近期到期付款。
✅
当前版本说明：本手册基于工具原型（v1.0）编写。原型使用示例数据，共包含10个项目（PROJ-001 至 PROJ-010）和5位示例项目经理。正式版本将对接公司实际数据系统。

7个模块总览
工具由7个相互联动的模块组成，每个模块专注于不同的分析视角。
🏠
全局总览
Dashboard
整个项目组合的"仪表盘"。显示核心 KPI、收支趋势图、即将到期付款、逾期预警，以及客户应收集中度分析。
📋
项目列表
Project List
所有项目的汇总表，显示项目编号、客户名称、所在地区、项目经理、状态和合同金额。支持多维度筛选。
📈
项目详情
Project Detail
单个项目的深度视图，包含该项目的收支走势图和完整的收支流水记录，可切换查看不同项目。
📒
收支明细
Transactions
全部收支流水的总台账，支持按项目、类型、方向、状态、日期和项目经理进行筛选，并可对所有列排序。
🔮
月度预测
Monthly Forecast
未来12个月的滚动现金流预测，以矩阵表格和柱状图双视图展示，帮助提前规划资金安排。
💰
应收账款
Accounts Receivable
追踪所有未收回款项，含账龄分析（30/60/90天以上）、DSO（应收天数）计算和客户集中度分析。
💳
应付账款
Accounts Payable
追踪所有应付未付款项，显示近期到期和已逾期的付款，帮助 PMO 和财务提前安排付款计划。
如何打开工具
1
打开主页（全局总览）
在浏览器中打开 index.html 文件，这是工具的入口页面（全局总览）。推荐使用 Chrome 或 Edge 浏览器以获得最佳显示效果。
2
通过导航栏切换模块
每个页面顶部均有导航栏，点击对应名称即可跳转到其他模块。各模块文件存放在 cashflow-prototype/GCF/ 目录下，分别对应独立的 HTML 文件。
3
选择项目经理（可选）
每个页面右上角均有"项目经理"筛选器，选择某位 PM 后，当前页面的所有数据将自动更新为该 PM 负责的项目范围。选择"全部"可恢复查看全局数据。

核心功能：项目经理筛选器
贯穿全部7个模块的核心功能，是 PMO 监控单个 PM 的主要工具。
什么是 PM 筛选器

PM 筛选器是一个下拉选择框，出现在工具每个页面的右上角。通过选择不同的项目经理姓名，可以将当前页面的所有图表、表格和汇总数据快速切换为该 PM 负责范围内的数据视图。

ℹ️
联动说明：筛选器的作用范围仅限于当前页面。切换到其他模块时，筛选状态会独立保存，不会自动同步。如需在另一个模块查看同一位 PM，请在该模块页面重新选择。
如何使用
1
找到筛选器
进入任意模块页面后，在页面右上角找到"项目经理"下拉框，默认显示"全部"。
2
选择目标 PM
点击下拉框，选择想要查看的项目经理姓名（原型中共有：张三、李四、王五、赵六、陈七）。
3
数据自动更新
页面所有内容（KPI 卡片、图表、表格）将立即更新，仅显示该 PM 名下项目的数据。金额汇总也会相应调整。
4
恢复全局视图
在下拉框中选择"全部"，即可恢复查看全部项目的汇总数据。
典型使用场景
场景一：月度 PM 复盘
每月月初，PMO 逐个选择每位 PM，在应收账款和全局总览页检查该 PM 的逾期情况和资金状况，对问题项目及时跟进。
场景二：快速响应汇报需求
当管理层临时询问某位 PM 的项目回款情况时，PMO 可在30秒内切换到该 PM 视图，导出或截图汇报。
⚠️
注意：月度预测模块的图表在切换 PM 后会重新渲染，如果图表短暂空白属于正常现象，稍等1-2秒即可恢复。

  • 通过 PM 筛选器切换不同项目经理，比较各 PM 的应收状况，识别回款绩效差异。
  • 月末使用收支趋势图回顾本月现金流状况，并与上月进行对比。

预测数据的来源

月度预测基于收支明细中所有状态为"已确认"或"待确认"的未来日期条目，按照条目的"到期日"或"日期"字段归入对应月份。实际入账以"日期"为准，预计到期以"到期日"为准。

⚠️
预测局限性：月度预测仅反映已录入系统的收支计划，若实际发生变化（如客户延迟付款、新增合同），需在收支明细中更新对应条目的日期和状态，预测数据才会同步更新。
年初至今（YTD）收入与成本进度

月度预测页面底部新增了 YTD 收入与成本进度表格，显示每个项目从今年1月1日至今日的实际收款和成本完成情况，帮助 PMO 快速判断各项目的资金健康程度。

字段定义如何解读
YTD（年初至今） 从今年1月1日至今日，状态为"已确认"的实际收付记录 预测和待确认数据不计入，反映真实已发生的金额
收入进度 % YTD 实际已收款 ÷ 项目合同总金额 反映当年回款完成度。进度条蓝色，数值越高说明收款越快
成本进度 % YTD 实际已发生成本 ÷ 项目预算总成本 反映成本消耗速度。进度条橙色，数值越高说明资金消耗越快
合同总额 该项目签约合同价值，作为收入进度的分母 来自项目基础数据
预算总成本 * 该项目全年预计总成本，作为成本进度的分母 当前为原型示例数据，正式版需对接公司实际预算系统
⚠️
关键风险信号：若某项目的成本进度 % 明显高于收入进度 %，说明成本消耗速度快于回款速度，该项目存在资金垫付压力，需重点跟进回款进度。

模块 6：应收账款
追踪所有尚未收回的款项，提供账龄分析、DSO计算和客户集中度视图。
核心指标说明
指标定义如何解读
DSO
应收天数		 Days Sales Outstanding，平均多少天能收回应收款，计算方式：(未收总额 ÷ 已开发票总额) × 时间段天数 DSO越低越好。DSO超过60天通常需要关注，超过90天需要主动跟款。
账龄分布
AR Aging		 按逾期时长将未收款项分组：0-30天、31-60天、61-90天、90天以上 "90天以上"部分占比过高说明坏账风险较大，需重点跟催。
客户集中度 各客户在全部未收款中的金额占比 某一客户占比超过50%意味着回款过度依赖单一客户，风险较高。
应收账款表字段说明
字段说明
发票编号对应收支明细中的条目编号（CE-XXX）
项目该笔应收所属项目
客户欠款方客户名称
发票金额原始发票金额（人民币）
开票日期该发票的开具日期
到期日客户承诺付款的截止日期
状态未收款（逾期/未逾期）或已收款
账龄（天）从开票日到今天的天数；若已逾期则显示逾期天数（标红）
是否已收款若对应收款记录已录入系统，则显示已收款及收款日期
ℹ️
重要提示：应收账款模块仅显示方向为"收入"的条目（发票和收款），不包含支出类记录。客户集中度图表仅统计尚未收回的发票，已对应收款的发票不计入。

如何使用
  • 每周检查应付账款模块，优先处理"已逾期"条目，避免因延迟付款影响供应商关系或产生违约金。
  • 查看"30天内到期"清单，提前向财务申请付款授权，确保资金按时到位。
  • 通过 PM 筛选器逐一检查各 PM 项目的应付情况，确认付款计划已落实。

项目数据（Project Data）
字段名类型必填说明
id KEY字符串是系统内部唯一编号，格式：PROJ-XXX
projectNo字符串是业务项目编号，格式：GPMS + 年月 + 序号
name字符串是项目名称，命名规则：P-客户简称-地区-类型-期次
client字符串是甲方客户全称
region枚举是CN / AR / CA / DE / TH（国家代码）
pm字符串是负责该项目的项目经理姓名
status枚举是active（进行中）/ completed（已完成）/ on-hold（搁置）
contractValue数字是合同总价值，单位：人民币（元）
收支数据（Cash Entry Data）
字段名类型必填说明
id KEY字符串是唯一编号，格式：CE-XXX
projectId FK字符串是关联的项目 ID（对应 PROJ-XXX）
date日期是收支发生日期，格式：YYYY-MM-DD
type枚举是invoice / receipt / expense / payment
subtype字符串否细分类型，如"合同首款"、"阶段款"、"材料费"

currency枚举是CNY / USD / EUR / ARS / CAD / THB

dueDate日期否预计收/付款截止日期，格式：YYYY-MM-DD
status枚举是confirmed / pending / overdue（见下页定义）
paidByEntryId字符串否仅用于应收模块：若已有对应收款记录，填写收款条目的 ID（CE-XXX）

状态定义 & 常见问题
收支条目状态（Status）定义
状态值显示名称含义触发条件
confirmed 已确认 款项已实际发生或已明确安排，数据可信 收款已到账；或付款已确认将按时执行
pending 待确认 计划中的收支，尚未最终确认 发票已开但客户未确认付款日期；计划中的支出尚未审批
overdue 逾期 已超过到期日但尚未完成的收或付 当前日期 > dueDate 且款项未收/未付
常见问题
Q：为什么全局总览的净现金流和收支明细的合计不一样？
A：全局总览中的净现金流 = 全部"收入方向"金额 − 全部"支出方向"金额，包括所有状态（已确认、待确认、逾期）。收支明细的合计受筛选器影响，若筛选器非"全部"则结果会不同。请确认收支明细的所有筛选器均已清空，再比对两个数字。
Q：月度预测里的数字和实际收支明细对不上？
A：月度预测使用的是"到期日"字段进行月份归属，而收支明细排序使用的是"日期"字段。两者可能对应不同的月份，这是正常现象。例如：一笔4月15日开具的发票，到期日是5月10日，则在预测中归入5月，在明细中按4月显示。
Q：切换 PM 筛选器后，图表变成空白了？
A：这是图表重新渲染时的正常现象，通常1-2秒后自动恢复。如果长时间不显示，请尝试刷新页面后重新选择 PM。
Q：应收账款里看不到某条发票？
A：应收账款模块只显示方向为"收入（in）"的条目。如果某条发票在收支明细中存在但应收账款看不到，请检查该条目的"方向"字段是否正确设置为"in"。
Q：多币种的金额如何换算？
A：原型中使用固定汇率进行折算，汇率数据预设在系统内（如 USD→CNY 约7.25）。正式版本将根据需求决定是使用固定汇率还是实时汇率接口。所有跨项目汇总计算均以 amountCny（人民币折算金额）字段为准。

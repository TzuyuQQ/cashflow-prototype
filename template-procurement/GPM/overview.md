# GPM 采购管理 PRD 总览

## 模块背景与目标

本模块面向内部人员（PM、采购、PMO），覆盖「采购计划发起 → 审批 → 采购计划执行 → ERP 对账 → 付款跟踪」全链路。核心目标是：让 PM 在 GPM 系统内完成采购计划提报和进度追踪，让采购人员在系统内完成任务交付确认，并通过与 ERP 的数据集成，实现采购应付款的可视化管理。

**系统边界：**
- **GPM（本模块）**：面向内部人员，负责采购计划、计划跟踪、审批、应付清单查看。
- **GPP（渠道/服务商门户）**：面向外部供应商，供应商在 GPP 中查看 PO、按付款节点发起付款申请，申请后流转至 GPM 进行内部审批。

## 用户角色与权限

| 角色 | 可访问页面 | 操作权限 |
|---|---|---|
| PM（项目经理） | 采购计划单列表、采购计划单（新建/编辑）、采购台账 | 新建/编辑草稿、提交审批、查看台账 |
| 采购负责人（任务责任人） | 我的采购任务、更新任务、采购台账 | 查看自己负责的任务、填写完成信息、上传交付物 |
| 审批人（直属上级/PMO） | 采购计划审批、供应商付款申请审批 | 通过/驳回采购计划、通过/驳回付款申请 |
| 采购 Leader | 采购计划审批（只读通知） | 审批通过后接收通知，无操作权限 |
| 财务/采购负责人 | PO 应付清单 | 查看、导出应付款数据 |

## 公共业务规则

1. **采购计划编号**：系统按编码规则自动生成，与 ERP 内采购计划编号相互独立。
2. **币种**：原币种 + 原币价格（两字段），另换算人民币价格（月度汇率，来自 ERP）。
3. **ERP 匹配**：通过「项目编号」粗匹配拉取对应 ERP 采购申请单列表；若申请单已填写「采购计划编号」（中国大区强制要求）则精确匹配；未填写时可在 GPM 侧手动维护关联，填入 PO Code 后自动带出其余 ERP 字段。废弃原「供应商名称 + 金额」一对一匹配方案。
4. **任务通知**：每周汇总一次，通知相关责任人「已逾期任务数 + 临近逾期任务数」。
5. **审批通知**：审批通过后给采购 Leader 发通知；若审批人本身即 PMO，则不重复通知 PMO。
6. **语言**：支持中英文切换。

**枚举值定义：**

| 枚举名 | 取值 |
|---|---|
| 订单类型 | 物料 / 资产 / 费用（待 ERP 侧确认最终枚举） |
| 任务状态 | 未开始 / 进行中 / 已完成 / 逾期未完成 |
| 采购计划任务颜色标记 | 绿色（已完成）/ 黄色（距计划结束日 ≤5 天未完成）/ 红色（已逾期未完成）/ 灰色（已完成含延期） |
| 付款支付状态 | Not Start / Pending / Closed |
| 采购跟进状态 | 进行中 / 已关闭（PO Code 有值时自动关闭） |
| 审批状态 | 待审批 / 已通过 / 已驳回 |

## 数据模型概览

| 数据对象 | 主要字段 |
|---|---|
| 采购计划单 | 采购计划编号、项目编号、项目名称、项目所在地、项目基本情况、申请采购内容、采购交付要求、采购预算、到货/入场时间、调试完成/撤场时间、**采购负责人**、审批状态、采购跟进状态、申请人、申请日期 |
| 采购计划任务 | 任务名称（固定4条）、计划完成时间、责任人、任务状态、交付物附件 |
| 采购附件 | 技术文件、合同模版、其他文件 |
| 中标信息 | 中标供应商、原币币种、原币价格、人民币价格、定标时间 |
| ERP 采购订单（PO） | PO Code、物料号、物料名称、物料分类、规格型号、采购数量、原币含税单价/总价、人民币含税总价、订单类型 |
| 供应商付款申请 | 付款申请单号、PO Code、付款节点、申请金额、发票附件、证明文件、审批状态 |

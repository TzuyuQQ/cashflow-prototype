# 采购管理 模块蓝图

## 模块概述

本模块面向项目经理（PM）和采购人员，覆盖「采购计划发起 → 审批 → 采购计划执行 → ERP 对账 → 付款跟踪」全链路。核心目标是：让 PM 在 GPM 系统内完成采购计划提报和进度追踪，让采购人员在系统内完成任务交付确认，并通过与 ERP 的数据集成，实现采购应付款的可视化管理。

**系统边界说明：**
- **GPM（本模块）**：面向内部人员（PM、采购、PMO），负责采购计划、计划跟踪、审批、应付清单查看。
- **GPP（渠道/服务商门户）**：面向外部供应商，供应商在 GPP 中查看与我方签订的采购订单（PO）、按付款节点发起付款申请。GPP 的付款申请提交后流转至 GPM 进行内部审批。

## 公共字段定义

### 枚举值

| 枚举名 | 取值 |
|---|---|
| 订单类型 | 物料 / 资产 / 费用（待 ERP 侧确认最终枚举，当前对应：物料→物料采购，资产→设备/固定资产，费用→服务/人力） |
| 任务状态 | 未开始 / 进行中 / 已完成 / 逾期未完成 |
| 采购计划任务颜色标记 | 绿色（已完成）/ 黄色（距计划结束日 ≤5 天，未完成）/ 红色（已逾期，未完成）/ 灰色（已完成，含延期已完成） |
| 付款支付状态 | Not Start / Pending / Closed |
| 采购跟进状态 | 进行中 / 已关闭（po code 有值时自动关闭） |

### 业务规则

- **采购计划编号**：系统按编码规则自动生成，与 ERP 内采购计划编号相互独立。
- **币种**：原币种 + 原币价格（两字段），另换算人民币价格（月度汇率，来自 ERP）。
- **ERP 匹配**：通过「项目编号」粗匹配拉取对应 ERP 采购申请单列表；若申请单已填写「采购计划编号」（中国大区强制要求）则精确匹配；未填写时可在 GPM 侧手动维护关联，填入 PO Code 后自动带出 ERP 字段。
- **ERP 关联层级**：采购申请单与 PO 为 1:N 关系；每条 PO 对应唯一供应商。废弃原三层顺序关联方案，改为通过项目编号/销售订单号直接关联。
- **任务通知**：每周汇总一次，通知相关责任人「已逾期任务数 + 临近逾期任务数」。
- **审批后通知**：审批通过后给采购 Leader 发通知；若审批人本身即 PMO，则不重复通知 PMO。
- **语言**：支持中英文切换（与现有资源系统一致）。

## 页面跳转关系

- **采购计划单列表** 点击「新建」→ 采购计划单（新建模式，弹窗确认立项状态）
- **采购计划单列表** 点击编号或「查看/编辑」→ 采购计划单（查看/编辑模式，跳过立项弹窗，直接进入详情）
- **采购计划单列表** 点击「查看台账」→ 采购台账（携带申请 ID，定位至对应行）
- **采购台账** 点击定标时间 / 采购合同/PO 签订列 → 更新任务页面（跳转至独立页面，填写完成信息）
- **我的采购任务** 点击「去完成」→ 更新任务页面（携带任务类型）
- **更新任务** 点击「返回列表」/ 取消 → 我的采购任务
- **更新任务** 提交完成 → 我的采购任务
- **供应商付款申请审批** 列表点击行 → 付款申请详情（含审批流查看）；数据来源为供应商在 GPP 提交的申请
- **供应商付款申请审批** 点击「审批通过 + 下推 ERP」→ 单点登录跳转 ERP 付款申请单页面（外部跳转，携带参数）
- **PO 应付清单（月度视图）** 点击行 → 自动切换至付款节点视图（携带 PO Code 筛选）

> **注意：页面跳转关系仅在此处文字描述，目录结构永远平铺，不因跳转关系产生嵌套。**

**GPP 页面跳转：**
- **采购订单列表** 点击行 → 采购订单详情
- **采购订单列表** 点击某付款节点「发起申请」→ 发起付款申请（携带 PO Code + 付款节点）
- **采购订单详情** 点击某付款节点「发起申请」→ 发起付款申请（携带 PO Code + 付款节点）
- **付款申请列表** 点击「发起付款申请」→ 发起付款申请（需选择 PO Code + 付款节点）
- **发起付款申请** 提交后 → 流转至 GPM「供应商付款申请审批」

## 页面详细说明

---

### 采购计划单列表

**功能描述：** 展示当前用户相关的所有采购计划记录，支持搜索筛选，可新建或进入详情。

**页面元素：**
- 顶部区域：搜索栏（采购计划编号、项目编号、项目名称、申请人）；「新建采购计划」按钮
- 主体区域：表格，每行一条采购计划
- 操作区域：每行提供「查看/编辑」「查看台账」操作入口

**字段说明：**

| 字段名 | 类型 | 说明 |
|---|---|---|
| 采购计划编号 | 文本 | 系统自动生成 |
| 项目编号 | 文本 | 可为空 |
| 项目名称 | 文本 | |
| 申请人 | 文本 | |
| 申请日期 | 日期 | |
| 审批状态 | 枚举 | 待审批 / 已通过 / 已驳回 |
| 采购跟进状态 | 枚举 | 进行中 / 已关闭 |

### 我的采购任务

**功能描述：** 任务责任人的统一任务入口，汇总自己负责的所有采购任务节点，支持按状态筛选，可直接进入"更新任务"页面完成任务。

**页面元素：**
- 顶部标签栏：全部 / 未完成 / 已逾期 / 已完成（含各状态数量角标）
- 主体区域：任务列表，每行一个任务节点

**字段说明：**

| 字段名 | 类型 | 说明 |
|---|---|---|
| 状态 | 色点 + 文字 | 颜色规则同公共枚举；未完成 / 临近截止 / 已逾期 / 已完成 |
| 任务名称 | 彩色标签 | 定标完成（橙）/ 合同签订（蓝）/ 技术文件（绿）/ 招采发起（紫） |
| 采购计划编号 | 文本 | |
| 项目名称 | 文本 | |
| 计划完成日期 | 日期 | |
| 实际完成日期 | 日期 | 未完成时显示「—」 |
| 操作 | 按钮 | 未完成时「去完成」；已完成时「查看」 |

**业务规则：**
- 已逾期行以浅红色背景高亮。
- 顶部导航 ToDo 角标展示当前未完成任务数。

---

### 更新任务

**功能描述：** 任务责任人填写任务完成信息的操作页面，支持上传交付物、填写实际完成日期。「定标完成」任务额外需填写中标信息。

**页面元素：**
- 顶部面包屑：采购管理 / 我的采购任务 / 更新任务
- Card 1：任务信息（只读）— 采购计划编号、项目名称、任务名称、计划完成日期、当前状态、责任人、交付物要求
- Card 2：填写完成信息 — 动态内容，根据任务类型展示
- 底部操作栏：返回列表 / 取消 / 确认完成

**任务类型对应表单：**

| 任务类型 | 额外字段 |
|---|---|
| 定标完成 | 中标供应商（必填）、定标时间（必填）、原币币种（必填）、原币价格（必填） |
| 招采发起 / 技术文件细化 / 采购合同/PO 签订 | 无额外字段 |
| 所有类型 | 实际完成日期（必填）、交付物上传（按任务类型标注）、备注（选填） |

**业务规则：**
- 实际完成日期早于计划日期：标记「已完成」；晚于计划日期：标记「已完成（含延期）」。
- 提交成功后返回我的采购任务列表，对应任务行状态更新。

---

### 采购计划单（新建/编辑）

**功能描述：** PM 填写采购计划，包含基本信息、采购计划任务、附件，提交后进入审批流。

**页面元素：**

- **Part 1：采购基本信息**
- **Part 2：采购计划**（任务列表）
- **Part 3：附件**
- **底部操作栏**：保存草稿 / 提交审批

**字段说明（Part 1）：**

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 采购计划编号 | 文本（只读） | 自动 | 系统自动生成，提交后锁定 |
| 项目编号 | 文本 | 否 | 非必填；若填写可自动带出项目名称、项目所在地 |
| 项目名称 | 文本 | 是 | |
| 项目所在地 | 树形选择（国家层级） | 是 | 与项目详情页交互方式一致；项目编号不存在时可手动选择 |
| 项目基本情况 | 多行文本 | 是 | 项目背景、概况 |
| 申请采购内容 | 多行文本 + 附件 | 是 | 采购范围、技术标准规范（含图纸等） |
| 采购交付要求 | 多行文本 + 附件 | 是 | 验收要求、质保要求、付款条件、罚则、备品备件等，可上传合同模版 |
| 采购预算 | 多行文本 + 附件 | 是 | 预算测算明细，文本填写 + 可上传附件（无固定模版） |
| 到货/入场时间 | 日期 | 是 | |
| 调试完成/撤场时间 | 日期 | 是 | |
| 采购负责人 | 人员选择 | 是 | 不可选发起人自己；选择后自动带入 Part 2「招采发起」「定标完成」「采购合同/PO 签订」三个节点的责任人（「技术文件」节点不带入） |

**字段说明（Part 2：采购计划任务）：**

固定 4 条任务（不可增删，责任人可选人）：

| 任务名称 | 默认责任部门 | 交付物要求 | 说明 |
|---|---|---|---|
| 技术文件 | 规划 | 必须上传技术文件（如 Part 3 技术文件已上传，则此任务自动标记为已完成） | |
| 招采发起 | 采购 | 上传招采邮件截图或招采文件 | |
| 定标完成 | 采购 | 完成时必填：中标供应商、原币价格、原币币种、定标时间 | 定标时间用于计算人民币换算的月度汇率 |
| 采购合同/PO 签订 | 采购 | 上传合同/PO 相关文件 | |

每条任务字段：

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 任务名称 | 文本（只读） | — | 固定 |
| 计划完成时间 | 日期 | 是 | |
| 责任人 | 人员选择 | 是 | 可多选 |

**字段说明（Part 3：附件）：**

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 技术文件 | 附件（多个） | 条件必填 | 若 Part 2「技术文件」任务状态为已完成，则此处必传；上传后自动将该任务标记为已完成 |
| 合同模版 | 附件（多个） | 否 | |
| 其他文件 | 附件（多个） | 否 | |

**审批流：**

```
申请人 → 上级审批
    ├── 通过：发代办任务给各责任人；通知采购 Leader；若审批人≠PMO 则抄送 PMO
    └── 驳回：仅通知申请人，无其他通知
```

**业务规则：**
- 审批通过后，方给 Part 2 各任务责任人发待办通知。
- 任务责任人完成任务时，需在「任务完成弹窗」上传对应交付物，方可标记完成。
- 定标完成任务完成时，需填写：中标供应商、原币币种、原币价格、定标时间。

### 采购台账

**功能描述：** 以采购计划为基础行，横向汇总采购计划任务完成状态（色卡）、中标信息、ERP 采购订单（PO）信息，供 PM 和管理人员全局掌握每条采购计划的全生命周期状态。

**页面元素：**
- 顶部区域：筛选栏（采购计划编号、项目编号、项目名称、申请人）
- 主体区域：横向宽表，分组列头（项目基本信息 / 采购计划 / 中标信息 / ERP 信息）
- 操作区域：各任务单元格可点击操作（责任人完成任务）；ERP 字段可手动维护 PO Code

**字段说明：**

*项目基本信息组：*

| 字段名 | 类型 | 说明 |
|---|---|---|
| 采购计划编号 | 文本 | |
| 项目编号 | 文本 | |
| 项目名称 | 文本 | |
| 项目所属大区 | 文本 | |
| 申请人 | 文本 | |
| 采购对接人 | 文本 | 任务责任人 |
| 项目交付地（国家） | 文本 | |
| 采购内容 | 文本摘要 | |
| 交付要求 | 文本摘要 | |
| 预算 | 金额 | |
| 到货/入场时间 | 日期 | |
| 采购计划相关文档 | 附件列表链接 | 点击展开列表，可查看/下载 |

*采购计划组（4 列，每列对应一个任务）：*

| 字段名 | 类型 | 说明 |
|---|---|---|
| 技术文件细化 | 日期 + 色卡 | 显示计划完成时间；颜色规则见公共枚举；悬停显示「已完成/逾期 N 天/距截止 N 天」 |
| 招采要求发送 | 日期 + 色卡 | 同上 |
| 定标时间 | 日期 + 色卡 | 同上 |
| 采购合同/PO 签订 | 日期 + 色卡 | 同上 |

*中标信息组：*

| 字段名 | 类型 | 说明 |
|---|---|---|
| 中标供应商 | 文本 | 定标任务完成时填入 |
| 原币价格 | 金额 + 币种 | 两字段：原币币种 + 原币价格 |
| 人民币价格 | 金额 | 原币 × 定标所属月汇率（来自 ERP） |

*ERP 信息组：*

| 字段名 | 类型 | 说明 |
|---|---|---|
| 采购计划单（ERP） | 文本 | 通过项目编号粗匹配 ERP 采购申请单；若申请单填写了「采购计划编号」则精确匹配；匹配不上可在 GPM 侧手动维护关联 |
| 物料号 | 文本 | 通过 PO Code 带出 |
| 物料名称 | 文本 | 通过 PO Code 带出 |
| 物料分类 | 文本 | 通过 PO Code 带出 |
| 规格型号 | 文本 | 通过 PO Code 带出 |
| 采购数量 | 数值 | 通过 PO Code 带出 |
| 原币含税单价 | 金额 | 通过 PO Code 带出 |
| 原币含税总价 | 金额 | 通过 PO Code 带出 |
| 人民币含税总价 | 金额 | 通过 PO Code 带出 |
| PO Code | 文本 | ERP 自动匹配；匹配不上时可手动填写 |
| 订单类型 | 枚举 | 物料 / 资产 / 费用 |
| 采购跟进状态 | 枚举 | PO Code 有值自动变为「已关闭」 |

**业务规则：**
- 任务色卡状态实时根据计划时间与当前日期计算。
- ERP 关联通过「项目编号」粗匹配拉取对应采购申请单；若申请单填写了「采购计划编号」（中国大区强制）则精确匹配；匹配不上时显示「匹配不上，请手动维护」提示，点击可手动填写 PO Code，填写后自动带出其余 ERP 字段。
- ERP 关联层级：采购申请单与 PO 为 1:N；每条 PO 对应唯一供应商；关联通过项目编号/销售订单号直接建立，不再使用三层顺序关联。
- ERP 字段存在反写逻辑：有 PO 订单时，项目编号、项目名称等字段可能由 ERP 回写。

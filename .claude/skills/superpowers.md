# Superpowers — Cash Flow Tool

Use this at the start of EVERY coding session or new feature.

## Workflow

### Step 1: Clarify (always run first)
Before any code, ask:
- What cash flow problem does this feature solve? (delayed payments, budget overruns, forecasting?)
- Who is the user? (project manager, finance team, contractor?)
- What does "done" look like for this session?

### Step 2: Spec
Write a mini-spec to findings.md:
- Input data (invoices, expenses, timesheets?)
- Output (chart, table, alert, export?)
- Edge cases (negative cash flow, missing data, multi-currency?)

### Step 3: Plan
Write numbered steps to task_plan.md before touching any file.
Include which component/file each step touches.

### Step 4: Execute
Follow the plan step by step.
Do NOT improvise. If requirements change, update task_plan.md first.

### Step 5: Review
After each feature:
- Does the dashboard reflect real-time data?
- Are all monetary values formatted consistently?
- Is error handling in place for missing/null values?
